# 用户端开发方案调整与 UI 问题检索

## 方案调整

- AI 客服模块调整到项目上传 GitHub 前集中开发。
- 当前阶段优先打磨基础查询闭环：首页检索、药品查询、药品详情、会员中心、药店信息。
- 用户端主导航暂不展示 AI 客服入口，避免用户误以为该功能已接入真实服务。

## 本轮问题检索

- 首页“常用分类”跳转携带 `category` 参数，但药品查询页未消费该参数，点击后实际筛选不生效。
- 首页和药品详情页存在 AI 客服强入口，不符合新的开发优先级。
- 用户端入口加载了全部 Element Plus 图标，主包体积偏大。
- 会员中心网络异常时可能显示偏技术化错误文案。
- AI 客服页仍是静态演示，容易与真实功能混淆。

## 已完成优化

- 首页改为“快速检索”，使用查询页已支持的 `type`、`isRx`、`stockStatus` 参数。
- 药品查询页支持从首页入口接收筛选参数。
- 顶部导航移除 AI 客服入口。
- 药品详情页“咨询此药”调整为“查看药店电话”。
- AI 客服页改为暂未开放说明页，并引导用户使用药品查询和药店信息。
- 前端图标改为页面按需导入，不再全量注册图标。
- 会员中心网络异常提示调整为更易懂的服务启动提示。

## 验证结果

- 已执行 `npm run build`，用户端生产构建通过。
- 构建主业务包体积相较上一轮下降；仍有 Element Plus 全量样式和组件带来的体积提示，后续可引入按需组件加载进一步优化。
