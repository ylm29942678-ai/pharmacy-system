乡镇药店轻量化管理系统 V1.0 - 综合说明书
(基于管理端核心页面功能基准)

=========================================
第一部分：V1.0 管理端页面核心功能清单
=========================================
[控制台与系统监控]
- 控制台页面：整合核心数据、快捷操作与业务提醒。展示当日营业额、订单总数、库存与近效期预警等核心运营数据，提供高频业务快捷入口及最新业务明细展示。
- 日志表页面：静默捕获并全量记录用户登录、增删改等核心动作，支持多条件筛选，保障业务可追溯，且仅管理员可见。

[基础档案管理]
- 药品信息页面：维护中药、西药、医疗器械的全生命周期信息。支持精细化剂型管理（如糖浆、瓶装等筛选），通过 status 字段实现停售药品的逻辑删除。
- 供应商页面：建立供货渠道档案，记录合作状态，保障采购溯源。
- 顾客/会员页面：登记顾客资料与会员等级，记录累计消费金额，支持启用/禁用会员身份。

[进销存核心业务]
- 库存页面：按药品+批号精细管理。展示库存数量、药柜位置、成本价与效期，并通过颜色区分醒目预警近效期或不足库存。
- 采购业务：在采购主表创建单据关联供应商。采购明细表作为入库依据，完整保留所购药品的批号、效期、进价等核心字段。
- 销售业务：主表集中展示订单状态、总价与支付方式。明细表展示对应单据药品的销售数量及单价，主表关联顾客信息以更新会员累计消费。
- 盘点表页面：记录账面库存与实际盘点数量。系统自动依据差额计算盈亏数量与金额，并要求填写盈亏原因留底。

[系统安全管理]
- 用户页面：管理员负责账号新增与权限分配。双角色权限控制结合前端动态渲染与后端校验，双重防范越权操作。

=========================================
第二部分：V1.0 核心业务流程说明
=========================================
本流程严格依据《管理端页面功能全》中的 V1.0 核心基础功能基准进行梳理。

1. 基础数据准备与建档流程
在系统正式流转前，需完成核心基础数据的初始化。
- 账号与权限配置：系统管理员分配账号并设定角色。
- 供应商与会员入库：建立供货渠道档案，并在顾客页面登记会员信息以支撑核验。
- 药品与分类精细化建档：录入全生命周期信息，药品表支持精细化的剂型管理（如快速定位特定剂型）。同时，分类数据维护可通过专属的 remark 字段大幅提升数据导入审核效率。

2. 采购入库业务流程
该流程规范药品进入药店体系的单据流转。
- 发起采购：操作员创建采购单并关联指定供货商。
- 录入采购明细留存依据：逐一添加进货药品，完整保留批号、生产日期、有效期等核心依据。（注：系统初期侧重单据记录查核）。

3. 销售出库业务流程
- 身份核验与快速开单：顾客结账时，通过控制台页面的快捷入口快速发起销售，并关联顾客页面核对会员信息。
- 录入销售明细：检索目标药品并生成订单明细。（注：初期版本重在单据流水固化与对账）。
- 结算与单据生成：确认订单总价并更新为已支付状态，形成完整数据并汇总至控制台工作台。

4. 库存盘点核对流程
- 动态监控与预警：基于页面颜色区分实时监控预警保质期与数量。
- 发起盘点：选择特定药品与批号，自动拉取当前账面库存。
- 账实核查与盈亏记录：录入实际库存后，系统自动计算并记录盈亏数量及金额，操作员需填写原因留存备注。

5. 系统安全与操作审计流程
- 包含基于角色的双重操作拦截与日志表全量静默捕获，记录完整操作场景。

第三部分：V1.0 基础部署说明书
=========================================
满足系统初期在单机或本地主机的快速上线需求。

1. 基础运行环境准备
需安装 JDK 17（Spring Boot 服务必需）、Node.js 18+（前端打包）、MySQL 8.0+ 以及 Nginx（Web 服务与反向代理）。

2. 数据库基础部署
- 创建数据库：字符集必须选择 utf8mb4，排序规则 utf8mb4_general_ci（兼容生僻中药名）。
- 数据初始化：导入 11 张核心表 SQL，并确保包含了药品剂型筛选项及分类表 remark 字段。

3. 后端服务编译与运行
- 执行 mvn clean package -DskipTests 命令打包。
- 使用 java -jar pharmacy-admin-1.0.0.jar --spring.profiles.active=prod 启动（生产环境可配合 nohup 后台运行）。

4. 前端服务编译与发布
- 执行 npm install 及 npm run build，将生成的 dist 文件夹移至目标部署目录。

5. Nginx 动静分离配置
- 配置 location / 代理静态页面，并使用 try_files 解决 Vue 路由刷新 404 问题。
- 配置 location /api/ 将接口请求 proxy_pass 反向代理至后端端口。

附录：迭代升级规划（随业务阶段逐步实施）
=========================================

4. 商业数据智能：引入可视化图表分析营业趋势，支持会员生日提醒与智能晋级规则等深度运营策略。
